## 🎨 README风格

### 语气
- ✅ 清晰简洁
- ✅ 专业但友好
- ✅ 面向使用者

### 格式
- ✅ 使用emoji增强可读性
- ✅ 代码块用正确的语法高亮
- ✅ 表格对齐整齐
- ✅ 链接完整有效

### 示例

**好的标题：**
```markdown
# Stock Simulation Trader v1

> 自动化股票模拟交易系统，支持策略回测和实时监控
```

**不好的标题：**
```markdown
# stock-trader
```

**好的功能描述：**
```markdown
## 功能特性

- ✅ **实时行情监控** - 支持20+交易所，延迟<1s
- ✅ **自动交易执行** - 基于策略自动买卖
- ✅ **风险控制** - 止损止盈，仓位管理
```

**不好的功能描述：**
```markdown
## 功能
- 监控
- 交易
- 风控
```

## 📞 问答

**Q: 每次都要写这么长的README吗？**

A: 不需要！
- **小脚本** - 用简化模板
- **正式项目** - 用完整模板
- **工具类** - 只写核心功能

**Q: README用什么语言写？**

A:
- **中文项目** - 用中文
- **开源项目** - 用英文
- **内部工具** - 用中文

**Q: 配置信息写在哪里？**

A:
- **README** - 只写配置说明
- **config.example.json** - 配置模板
- **敏感信息** - 用环境变量
