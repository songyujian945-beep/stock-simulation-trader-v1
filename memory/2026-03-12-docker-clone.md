# 分身计划 - Docker + 本地AI

## 目标
在Docker中运行一个本地AI模型，作为我的"分身"

## 技术方案

### 方案1：Ollama（推荐）
```bash
# 安装后运行
docker run -d -p 11434:11434 ollama/ollama
ollama run qwen2:7b  # 或其他模型
```

优点：
- ✅ 简单易用
- ✅ 支持中文（Qwen、GLM等）
- ✅ 社区活跃

### 方案2：LocalAI
```bash
docker run -p 8080:8080 localai/localai:latest
```

### 方案3：vLLM（高性能）
需要GPU支持

## 分身能力对比

| 模型 | 大小 | RAM需求 | 中文能力 |
|------|------|---------|----------|
| Qwen2-7B | 4.7GB | 8GB+ | ✅ 优秀 |
| GLM-4-9B | 6GB | 12GB+ | ✅ 优秀 |
| Llama3-8B | 5GB | 10GB+ | ❌ 一般 |
| Phi-3-mini | 2GB | 4GB+ | ❌ 一般 |

## 分身 vs 我

| 能力 | 我（云端） | 分身（本地） |
|------|-----------|-------------|
| 响应速度 | 网络依赖 | 快（本地） |
| 离线使用 | ❌ | ✅ |
| 智能程度 | 高 | 中等 |
| 成本 | 按token计费 | 免费（电费） |
| 隐私 | 数据上传 | 完全本地 |

## OpenClaw连接本地分身

修改配置：
```json
{
  "agents": {
    "defaults": {
      "model": "ollama:qwen2:7b"
    }
  }
}
```

## 分身可以做什么

1. ✅ 简单对话
2. ✅ 代码辅助
3. ✅ 文本处理
4. ❌ 复杂推理
5. ❌ 工具调用（需要额外配置）
